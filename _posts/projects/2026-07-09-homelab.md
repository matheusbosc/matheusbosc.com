## How It Started

It started small — Jellyfin, set up to stop manually organizing my media library by hand. For a while, that was the whole homelab. One service, doing one job.

Then I tore it all down.

## Starting Over, Properly

I made the call to install Proxmox VE, mainly so a crash in one service couldn't take the whole setup down with it. Once that foundation was in place, I rebuilt from scratch:

- **Jellyfin** came back first, same job as before
- **Portainer** followed, to make spinning up new Docker containers far less painful
- **Vaultwarden** came next — I'd been using 1Password, but ran into issues keeping it updated (possibly a trial that lapsed, honestly don't remember exactly why). Self-hosting my own password manager solved that permanently.

From there, it snowballed the way homelabs tend to:

- **Jellyseerr** and the ***arr* suite** joined to round out the media side
- I started hosting my own dev projects too — including the servers behind [Chat++](/projects/chat-plus-plus) and my [TODO app](/projects/todo-app)
- Game servers followed too, for whatever I'm playing with friends at the time

## Networking & Security

Once the homelab had more than a couple of services running, keeping them isolated and secure became its own project:

- **Network segmentation** — I split services across VLANs and Docker networks, so nothing on one part of the network can interfere with (or compromise) another.
- **Secure remote access** — instead of exposing ports directly, I set up Cloudflare Tunnels with a zero-trust access model, so I can reach my services from anywhere without opening up my home network to the internet.
- **Resource optimization** — with limited hardware, I leaned on virtualization and Docker to squeeze as much out of what I have as possible, rather than running everything bare-metal.

## Monitoring & Automation

The homelab isn't just "set it and forget it" — it's fully automated where it can be, and I keep an eye on it where it can't:

- CI/CD pipelines handle deployment and updates for the services and projects I host myself
- Logging and monitoring give me visibility into what's actually happening across the stack, instead of finding out something broke after the fact

## The Hardware

None of this runs on anything fancy. My main server is an old HP laptop — low on RAM and storage, but it's never given me a real problem, and it's been running solid the whole time.

Alongside it, I repurposed an old Asus Chromebox that was just sitting around the house, unused. It's now running Home Assistant OS, powering a dashboard for my bedroom.

## What I've Learned

This homelab has been less about any one service and more about the practice of running infrastructure myself:

- Real, hands-on experience with network security and service orchestration — the kind of thing that's hard to learn without actually breaking something first
- A better sense of the trade-off between simplicity and flexibility — every homelab decision is a balance between "easy to maintain" and "does exactly what I want"
- Practice with monitoring, logging, and troubleshooting in an environment where I'm the only one responsible for fixing it

It's not a big setup, but it works — and I'm hoping to keep expanding it over time.
