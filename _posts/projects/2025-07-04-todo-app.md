## The Assignment

CS50 ends with a final project — build something real, using what the course actually taught. No fixed spec, just a requirement to show you could put the pieces together yourself. I went with something practical: a TODO app.

## What I Built

It wasn't just a basic task list. Alongside the standard CRUD features, I built:

- A full **user system** — accounts, authentication, the works
- **Shared task boards**, so users could collaborate on the same list instead of just managing their own

The whole thing ran on **Python and Flask**, which was also my first real project outside a game engine — no Unity UI, no editor, just code and a keyboard. That alone made it feel like a different kind of building.

## The Hard Part

The sharing system was where things got genuinely tricky. Basic CRUD is one thing — a single user managing their own data is a fairly contained problem. But once multiple users needed to see and edit the *same* board, a few new problems showed up:

- Structuring the database so boards could belong to multiple users, not just one
- Making sure permissions and access were actually enforced, not just assumed
- Keeping the logic clean enough that adding "sharing" didn't turn into a tangle of special cases layered on top of the basic task features

Getting that right took more thought than anything else in the project — but it's also what made the app feel like something more than a tutorial clone.

## What It Led To

The TODO app was a one-off for a course, and once it was submitted, I went back to game dev for a while. But it planted something: building outside of a game engine, writing straight code with nothing but a keyboard, felt good in a way I hadn't expected. That itch eventually led to [Chat++](/projects/chat-plus-plus) — a much bigger step into general software development.
